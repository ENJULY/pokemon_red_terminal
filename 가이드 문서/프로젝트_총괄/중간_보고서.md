# 프로젝트 중간보고서 — Pokemon Red Terminal Edition

> Windows 콘솔(CMD / Windows Terminal)에서 ANSI 이스케이프 + UTF-8 반블록(▀▄) 문자로
> 동작하는 1세대 포켓몬스터 레드의 C++17 클론.
> **목표 범위: 팔레트시티 ~ 회색시티 1관장(브록) 클리어**

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | Pokemon Red - Terminal Edition |
| 개발 언어 | C++17 |
| 빌드 도구 | MinGW-w64 g++ (`build.bat` 자동 빌드 스크립트) |
| 대상 플랫폼 | Windows 10 / 11 (CMD 또는 Windows Terminal) |
| 렌더링 방식 | ANSI 이스케이프 + UTF-8 반블록 문자(▀▄) 더블 버퍼 |
| 자산 출처 | pret/pokered (Pokemon Red 공식 디스어셈블리) |
| 자산 변환기 | Python 스크립트 6종 (`tools/gen_*.py`) |
| 코드 규모 | C++ 약 9,500 LOC · Python 약 4,500 LOC |

## 2. 사용자 정의 클래스 (6개)

요구사항: **2개 이상** → 본 프로젝트는 **6개의 사용자 정의 클래스** 보유.
각 클래스는 `.h` (선언) + `.cpp` (구현) 로 분리되어 있다.

| # | 클래스 | 헤더 | 구현 | 역할 |
|---|--------|------|------|------|
| 1 | `Renderer` | `src/engine/renderer.h` | `src/engine/renderer.cpp` | ANSI 더블버퍼 렌더링 |
| 2 | `Input`    | `src/engine/input.h`    | `src/engine/input.cpp`    | Win32 콘솔 키 입력 폴링 |
| 3 | `Audio`    | `src/engine/audio.h`    | `src/engine/audio.cpp`    | MCI 기반 BGM/SE 재생 |
| 4 | `Game`     | `src/game/game.h`       | `src/game/game.cpp`       | Scene 상태머신, 게임 진입점 |
| 5 | `Overworld`| `src/game/overworld.h`  | `src/game/overworld.cpp`  | 맵 이동·워프·인카운터 |
| 6 | `Battle`   | `src/game/battle.h`     | `src/game/battle.cpp`     | 턴제 배틀 엔진 |

> 보조 데이터 타입(`struct`)으로 `Cell`, `Pokemon`, `Player`, `MapDef`,
> `NpcDef`, `TrainerDef`, `WarpDef`, `OwState`, `BattleState`, `SignDef` 등이 존재.

## 3. 객체지향 요소 정리표

> 본 프로젝트가 사용한 OOP/C++ 기능을 정리한 표.

| 요소 | 사용 위치 | 설명 |
|------|-----------|------|
| **클래스 (class)** | `Renderer`, `Input`, `Audio`, `Game`, `Overworld`, `Battle` | 6개의 사용자 정의 클래스 |
| **헤더/소스 분리** | 모든 클래스 | `.h` 선언, `.cpp` 구현 분리 |
| **캡슐화 (private)** | `Game::scene_`, `Battle::state_`, `Overworld::state_`, `Renderer::curr_/prev_` | 내부 상태는 private, 접근은 public 메서드로 |
| **함수 오버로딩 (Overloading)** | `Renderer::setCell` vs `setCellU` / `print` vs `printW` vs `printRaw` / `Battle::startWild` vs `startTrainer` vs `startBrock` | 같은 행위지만 인자/타입이 다른 함수 |
| **연산자 오버로딩 (암묵)** | `Pokemon p = {};` 초기화, `state_ = {};` 리셋 | 집합 초기화 (aggregate initialization) |
| **디폴트 인자 (Default argument)** | `setCell(...,const std::string& color = "")`, `startTrainer(..., int introId = 0)` | 함수 호출 단순화 |
| **생성자 (Constructor)** | `Game()`, `Overworld(Renderer&, Player&)`, `Battle(Renderer&, Player&)` | 객체 초기화 |
| **멤버 초기화 리스트** | `Battle::Battle(Renderer& r, Player& pl) : ren_(r), pl_(pl), state_{} {}` | 참조 멤버 초기화 |
| **참조 전달 (`&`)** | `Overworld(Renderer& r, Player& pl)` | 복사 없이 객체 공유 |
| **const 멤버 함수** | `eventData() const`, `wildSpeciesId() const`, `idx(int,int) const` | 객체 상태 불변 보장 |
| **정적 멤버 (static)** | `Input::poll()`, `Audio::playBGM()`, `Game::INTRO_LINES[]`, `Game::INTRO_COUNT` | 인스턴스 없이 호출, 클래스 공용 상수 |
| **constexpr 상수** | `Player::MAX_CUT_TREES = 16`, `MOVE_CUT = 18` | 컴파일 타임 상수 |
| **inline 함수** | `playerHasCut(...)`, `isTreeCut(...)`, `recalcStats(...)`, `firstAlive(...)` | 헤더 정의, 호출 비용 절감 |
| **enum class (강타입 열거형)** | `Scene`, `Key`, `OwEvent`, `BattleType`, `BattlePhase`, `BattleResult`, `CutsceneType`, `Type` | 타입 안전 enum |
| **구조체 + 메서드** | `OwState`, `BattleState`, `Cell` | POD 데이터 묶음 |
| **포인터 멤버** | `Game::battle_` (`Battle*`), `Game::ow_` (`Overworld*`) | 동적 생성/Lazy init |
| **참조 멤버** | `Battle::ren_` (`Renderer&`), `Battle::pl_` (`Player&`) | 외부 객체 참조 |
| **상태 패턴 (디자인 패턴)** | `Game::scene_` + `Scene::*` switch | Scene 별 update/render 분기 |
| **이벤트 큐 패턴** | `OwEvent`, `Overworld::popEvent()` | Overworld → Game 통신 |

> **상속/추상 클래스/가상 함수는 사용하지 않음.**
> 이유: Scene이 6개, Battle Phase가 14단계 등 데이터 기반 상태 머신이 더 단순하다고 판단.
> 향후 트레이너 AI 다형화 시 `class TrainerAI`의 추상 베이스로 확장 여지 있음.

### 4.3 핵심 클래스 상세

#### 4.3.1 `Renderer` — 더블 버퍼 ANSI 렌더링

```cpp
class Renderer {
public:
    int width  = 0;
    int height = 0;

    void init();                                                   // 콘솔 ANSI 활성화 + 크기 설정
    void setCell(int x, int y, char ch, const std::string& color = "");      // 1바이트 셀
    void setCellU(int x, int y, const std::string& utf8, const std::string& color = ""); // 멀티바이트 UTF-8 셀
    void print(int x, int y, const std::string& text, const std::string& color = "");
    void printW(int x, int y, const std::wstring& text, const std::string& color = ""); // 한국어 직접 출력
    void printRaw(int x, int y, const char* utf8_ansi);            // 타일 art ANSI 파싱
    void drawBox(int x, int y, int w, int h, const std::string& color = "");
    void fillRect(int x, int y, int w, int h, char ch, const std::string& color = "");
    void flush();        // diff 기반 출력 (변경된 셀만)
    void clear();        // 버퍼 공백 초기화
    void redrawAll();    // 강제 전체 재렌더

private:
    std::vector<Cell> curr_, prev_;   // 더블 버퍼
    void moveCursor(int x, int y);
    void enableAnsi();
    void hideCursor();
    int  idx(int x, int y) const { return y * width + x; }
};
```

**핵심 기능**:
- `setCell` (ASCII 1바이트) ↔ `setCellU` (UTF-8 멀티바이트) **오버로딩**
- `print` / `printW` / `printRaw` **오버로딩**으로 다양한 텍스트 입력
- `flush()` 가 `curr_` 와 `prev_` 를 비교해 **변경 셀만** ANSI 커서이동 + 출력 → 깜빡임 제거

#### 4.3.2 `Game` — Scene 상태 머신

```cpp
enum class Scene {
    INTRO, NAME_INPUT, RIVAL_NAME_INPUT,
    LAB_INTRO, STARTER_SELECT, RIVAL_INTERCEPT, RIVAL_BATTLE,
    RECEIVE_DEX,
    OVERWORLD,
    WILD_BATTLE, TRAINER_BATTLE, BOSS_BATTLE,
    POKEMON_CENTER, MART_EVENT,
    GAME_OVER, ENDING,
    WARP_MENU,
};

class Game {
public:
    Game();                  // 플레이어 초기화 + RNG seed
    void run();              // 메인 루프 (60 FPS)
    Renderer renderer;
    bool running = true;

private:
    Scene      scene_   = Scene::INTRO;
    int        frame_   = 0;
    Player     player_  = {};
    Battle*    battle_  = nullptr;     // 동적 생성 (lazy)
    Overworld* ow_      = nullptr;

    void changeScene(Scene next);
    void update(Key key);
    void render();
    void renderKorean();
    // ... 씬 별 update/render 함수 16쌍
};
```

**메인 루프**:
```cpp
void Game::run() {
    renderer.init();
    while (running) {
        Key key = Input::poll();
        update(key);              // 현재 Scene 분기
        renderer.clear();
        render();
        renderKorean();
        renderer.flush();
        Sleep(62);                // ~16 FPS
    }
}
```

#### 4.3.3 `Battle` — 턴제 배틀 엔진

```cpp
class Battle {
public:
    Battle(Renderer& r, Player& pl);

    // 함수 오버로딩: 야생 / 트레이너 / 보스 진입
    void startWild(int speciesId, int level);
    void startTrainer(const wchar_t* name, const wchar_t* preText,
                      int* partyIds, int* partyLevels, int partySize,
                      int introId = 0);    // ← 디폴트 인자
    void startBrock();

    void update(Key key);
    void render();
    void renderKorean();

    BattleResult result() const { return state_.result; }
    bool isDone() const { return state_.phase == BattlePhase::DONE; }

private:
    Renderer& ren_;          // 참조 멤버
    Player&   pl_;
    BattleState state_;

    void applyDamage(Pokemon& a, Pokemon& d, int moveId,
                     bool& superEff, bool& notEff, bool& noEff, int& damage);
    void grantExp();
    void checkLevelUp(Pokemon& p);
    int  chooseEnemyMove();          // 적 AI: 데미지 기댓값 최대 기술 선택
    void drawSprite(int x, int y, int speciesId, bool back);
};
```

**데미지 공식 (Gen1 단순화)**:
```cpp
damage = ((2 * level / 5 + 2) * power * atk / def) / 50 + 2
       * type_effectiveness * stab * critRand;
```

#### 4.3.4 `Overworld` — 맵 이동 시스템

```cpp
class Overworld {
public:
    Overworld(Renderer& r, Player& pl);
    void init();
    void update(Key key);
    void render();
    void renderKorean();

    OwEvent popEvent();                          // 이벤트 큐 → Game에 전달
    int     eventData() const;
    void onReturnFromBattle(bool won);

    void startOakIntercept();                    // OW 컷씬: 풀숲에서 오박사 등장
    void startRivalBlock();                      // OW 컷씬: 라이벌이 출구 막음

private:
    Renderer& ren_;
    Player&   pl_;
    OwState   state_;

    void tryMove(int dx, int dy);                // 이동 시도 + 워프/절벽/풀숲 검사
    bool checkTrainerSight();                    // 트레이너 시야 충돌
    void checkWarps(int x, int y);               // 워프 타일 → 페이드 트랜지션
    void triggerEncounter();                     // 야생 인카운터 추첨
    MapDef* curMap() const;
};
```

**이벤트 큐 패턴**: `Overworld::popEvent()` → `Game::update()` 가 이벤트별로 `changeScene(Scene::WILD_BATTLE)` 등 처리.

| 키 | 매핑 | 사용처 |

### 5.4 시각 디자인 원칙

- **색상**: ANSI 256 색 팔레트, pokered 원본 GB 4단계 → 5단계 회색조 (`255 / 250 / 244 / 232`)
- **타일 단위**: 8×8 pixel atom → 4개로 16×16 step → 32×32 block
- **반블록(▀▄)** 사용: 한 콘솔 셀에 위/아래 2픽셀 표현 → 픽셀 종횡비 1:1 유지
- **한국어 텍스트**: `printW` 로 wide string 직접 출력 (한 글자 = 2 셀)

### 6.2 Scene 상태 전이도

```
INTRO -> NAME_INPUT -> RIVAL_NAME_INPUT -> OVERWORLD (자기 집 2층 깨어남)
                                              |
   +------------------------------------------+
   |   (Overworld 내에서 다양한 이벤트 발생)
   v
   +--> 풀숲 진입(오박사 인터셉트) -> LAB_INTRO -> OVERWORLD(연구소)
```

### 7.1 ANSI 더블 버퍼 렌더링

```cpp
// engine/renderer.h
struct Cell { std::string ch = " "; std::string color = ""; };

class Renderer {
private:
    std::vector<Cell> curr_, prev_;     // 더블 버퍼

public:
    void flush() {                       // diff 기반 출력
        for (int y = 0; y < height; y++) {
            for (int x = 0; x < width; x++) {
                int i = idx(x, y);
                if (curr_[i].ch == prev_[i].ch && curr_[i].color == prev_[i].color)
                    continue;            // 변경 없음 → skip
                moveCursor(x, y);
                printf("%s%s%s", curr_[i].color.c_str(),
                       curr_[i].ch.c_str(), Color::RESET);
                prev_[i] = curr_[i];
            }
        }
        fflush(stdout);
    }
};
```

**효과**: 화면 전체 재출력 대비 95% 이상 출력량 감소 → CMD 깜빡임 제거.

```cpp
// game/player.h
static constexpr int MOVE_CUT = 18;

inline bool playerHasCut(const Player& pl) {
    for (int i = 0; i < pl.partySize; i++) {
        const Pokemon& p = pl.party[i];
        if (!p.species) continue;
        for (int j = 0; j < p.numMoves; j++)
            if (p.moves[j].moveId == MOVE_CUT) return true;
    }
    return false;
}

inline bool isTreeCut(const Player& pl, int mapId, int x, int y) {
    for (int i = 0; i < pl.cutTreeCount; i++)
        if (pl.cutTreeMapId[i] == mapId &&
            pl.cutTreeX[i] == x && pl.cutTreeY[i] == y) return true;
    return false;
}
```

## 9. 개발 중 발생한 장애요인과 문제 해결 과정

### 9.1 ANSI 더블 버퍼 깜빡임

| 항목 | 내용 |
|------|------|
| 문제 | `printf` 로 매 프레임 전체 화면을 다시 그리니 CMD에서 깜빡임 |
| 원인 | CMD가 ANSI 시퀀스를 한 줄씩 처리 → 부분 렌더 상태가 짧게 보임 |
| 해결 | `Cell` 더블 버퍼(`curr_`/`prev_`)로 diff 출력 (변경된 셀만 cursor 이동 후 출력) |
| 효과 | 출력량 90%+ 감소, 깜빡임 제거 |

### 9.2 한국어 출력 깨짐

| 항목 | 내용 |
|------|------|
| 문제 | `wprintf` 가 CMD 기본 codepage(949)에서 garbled |
| 원인 | console output CP가 UTF-8 아님 |
| 해결 | `SetConsoleOutputCP(CP_UTF8)` + `setlocale(LC_ALL, ".UTF-8")` + `printW(wstring)` 직접 출력 |
| 효과 | 모든 한국어 텍스트 정상 출력 |

### 9.3 build.bat 다른 PC에서 실행 실패

| 항목 | 내용 |
|------|------|
| 문제 | 처음에 절대경로(`C:\mingw64\bin\g++.exe`)로 작성 → 다른 팀원 PC에서 실패 |
| 원인 | MinGW 설치 경로가 사용자마다 다름 (`C:\msys64\mingw64`, `C:\TDM-GCC-64` 등) |
| 해결 | (1) PATH의 g++ 우선 탐색, (2) 7곳 표준 설치 경로 fallback, (3) `chcp 65001` 로 UTF-8 강제, (4) 발견 시 PATH에 임시 등록 |
| 효과 | 클린 Windows + MinGW 설치만 되어 있으면 무조건 동작 |

### 9.4 build.bat 인코딩 + 라인 엔딩

| 항목 | 내용 |
|------|------|
| 문제 | 한국어 메시지 포함 시 cmd.exe 파싱 오류 / `^` 줄 연속이 LF 라인엔딩에서 안 먹음 |
| 원인 | cmd.exe 가 LF 만 있는 .bat 의 `^` 토큰을 인식 못 함 |
| 해결 | (1) 모든 메시지 ASCII 만 사용 (2) `sed -i 's/$/\r/'` 로 CRLF 변환 |
| 효과 | 라인 연속 정상 동작 |

### 9.5 표지판 좌표 어긋남

| 항목 | 내용 |
|------|------|
| 문제 | pokered 원본 bg_event 좌표 그대로 사용 → A키 눌러도 반응 없음 |
| 원인 | 자동 생성된 우리 맵은 pokered와 좌표 체계가 다름 (스텝 단위 vs 픽셀 단위) |
| 해결 | Python 스크립트로 우리 맵의 실제 `=`, `O`, `R`, `S`, `+` 문자 좌표를 스캔 후 그 좌표 사용 |
| 효과 | 28개 표지판 모두 정상 트리거 |

### 9.6 상록숲 나무 밑동 렌더링

| 항목 | 내용 |
|------|------|
| 문제 | Forest 타일셋의 `$/%/&/(/)` 가 작은 그루터기 atom 만 그림 |
| 원인 | Forest tileset이 별도의 atom 인덱스를 사용 |
| 해결 | `tiles.h` 의 mapId 분기에서 MAP_4 의 `$/%/&/(/)` 를 모두 `TILE_OW3_C036` (overworld 큰 나무) 로 반환 |
| 효과 | 다른 맵과 같은 큰 나무 비주얼 |

### 9.7 플레이어 오른쪽 향함 스프라이트

| 항목 | 내용 |
|------|------|
| 문제 | 오른쪽 방향 입력 시 왼쪽 향한 스프라이트가 나옴 |
| 원인 | pokered red.png 가 LEFT idle/walk 만 제공 (RIGHT는 mirror해서 사용) |
| 해결 | (1) `tools/gen_sprites.py` 에 mirrored frame generator 추가 (`x' = 15 - x`) (2) `playerFrameIdx` 매핑에 F6/F7 추가 |
| 효과 | 4방향 모두 자연스러운 스프라이트 |

### 9.8 워프 후 벽에 박힘

| 항목 | 내용 |
|------|------|
| 문제 | 회색시티 체육관 워프 후 캐릭터가 벽 안 |
| 원인 | `destX=-1` 일 때 southEntry fallback (4, 0) 이 벽 셀임 |
| 해결 | 워프 destX/destY 를 (4, 12) 로 명시 (체육관 문 바로 위) |
| 효과 | 워프 후 정상 위치 |

### 9.9 미구현 건물 진입 시 무반응

| 항목 | 내용 |
|------|------|
| 문제 | 일부 건물(나니와 닉네임 평가소 등) 문이 destMap=-1 → 워프 시 크래시 또는 무반응 |
| 원인 | 9개 건물 내부가 미구현 |
| 해결 | `destMap == -1` 감지 시 "문이 잠겨있다..." 대화 트리거 |
| 효과 | 사용자에게 의도된 안내 메시지 |

### 9.10 Battle 인스턴스 메모리

| 항목 | 내용 |
|------|------|
| 문제 | `static` 멤버 변수가 두 번째 배틀에서 이전 상태를 유지 |
