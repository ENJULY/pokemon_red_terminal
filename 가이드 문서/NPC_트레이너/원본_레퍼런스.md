# 원본 NPC 배치 레퍼런스 (pokered)

> **이 문서가 무엇인가?**
> Pokemon Red 원본 디스어셈블리(`pret/pokered`)의 각 맵 NPC 배치 데이터를
> 우리 프로젝트 좌표 체계로 정리한 **참고 자료**.
>
> NPC 추가 작업 시 이 표를 보고 "원본은 여기에 누구를 두었는지" 확인 → 한국어 대사
> 작성 → `map_data.h`에 `NpcDef` / `TrainerDef` 항목 추가하면 됨.
>
> **원본 파일 위치**: `pokered_assets/data/maps/objects/*.asm`
> (`PalletTown.asm`, `Route1.asm`, `ViridianCity.asm`, `Route2.asm`,
> `ViridianForest.asm`, `PewterCity.asm`, `OaksLab.asm`, `PewterGym.asm`,
> `RedsHouse1F.asm`)


## 1. 사용법 (3 step)

1. 추가하고 싶은 맵을 찾는다 → 아래 §3 표에서 원본 NPC 좌표/스프라이트 확인.
2. 우리 `NPC_추가_가이드.md` 의 `NpcDef` 형식으로 변환 (좌표 그대로 사용 가능 — 우리 맵 폭이 원본과 동일).
3. 한국어 대사 작성 → `map_data.h` 의 해당 `MAP_N` 의 `npcs[]` 배열에 추가.

원본 .asm 한 줄 예시:
```asm
object_event 13, 20, SPRITE_YOUNGSTER, WALK, ANY_DIR, TEXT_VIRIDIANCITY_YOUNGSTER1
```
↓ 변환
```cpp
// MAP_VIRIDIAN (40×36) 의 npcs[] 배열에 추가
{13, 20, {
    L"청년: 어이! 너도 모험 중이야?",     // 한국어 대사 직접 작성
    L"청년: 풀숲 조심해. 야생 포켓몬 많아!",
    nullptr,
}, NPC_SPR_LITTLE_BOY},  // YOUNGSTER → 우리 스프라이트 매핑 (§2 참고)
```


## 2. 스프라이트 매핑 (pokered SPRITE_* → 우리 NPC_SPR_*)

우리는 13종 스프라이트만 보유 (`sprites.h`):

| pokered 원본 (SPRITE_*) | 우리 enum | 외형 비고 |
|--------------------------|-----------|----------|
| SPRITE_OAK         | `NPC_SPR_OAK`       | 박사 풀바디 |
| SPRITE_MOM         | `NPC_SPR_MOM`       | 여성 어른 |
| SPRITE_BLUE        | `NPC_SPR_BLUE`      | 라이벌 |
| SPRITE_GIRL        | `NPC_SPR_GIRL`      | 어린 소녀 |
| SPRITE_BRUNETTE_GIRL | `NPC_SPR_BRUNETTE` | 갈색머리 여성 |
| SPRITE_LITTLE_BOY  | `NPC_SPR_LITTLE_BOY` | 어린 소년 |
| SPRITE_LITTLE_GIRL | `NPC_SPR_LITTLE_GIRL` | 어린 소녀 |
| SPRITE_FISHER      | `NPC_SPR_FISHER`    | 낚시꾼 |
| SPRITE_GENTLEMAN   | `NPC_SPR_GENTLEMAN` | 정장 신사 / 과학자 대용 |
| SPRITE_GAMBLER     | `NPC_SPR_GENTLEMAN` (대용) | — 전용 없음 |
| SPRITE_GAMBLER_ASLEEP | `NPC_SPR_GENTLEMAN` (대용) | — 전용 없음 |
| SPRITE_YOUNGSTER   | `NPC_SPR_LITTLE_BOY` (대용) | — 청년 전용 없음 |
| SPRITE_COOLTRAINER_M | `NPC_SPR_GENTLEMAN` (대용) | — 트레이너 |
| SPRITE_COOLTRAINER_F | `NPC_SPR_BRUNETTE` (대용) | — 트레이너 |
| SPRITE_SUPER_NERD  | `NPC_SPR_GENTLEMAN` (대용) | — 안경 캐릭터 |
| SPRITE_NURSE       | `NPC_SPR_NURSE`     | 간호사 조이 |
| SPRITE_CLERK       | `NPC_SPR_CLERK`     | 마트 점원 |
| SPRITE_GRAMPS      | `NPC_SPR_GRAMPS`    | 노인 |
| SPRITE_POKE_BALL   | `NPC_SPR_POKE_BALL` | 바닥 아이템 (몬스터볼) |
| SPRITE_POKEDEX     | `NPC_SPR_GENTLEMAN` (대용) | 책상 위 도감 |
| SPRITE_GYM_GUIDE   | `NPC_SPR_GENTLEMAN` (대용) | 체육관 가이드 |
| SPRITE_SCIENTIST   | `NPC_SPR_GENTLEMAN` (대용) | 연구원 |

> **대용 사용 시 외형은 비슷한 것으로 매핑됨.** 본 프로젝트 범위에서는 추가 스프라이트 생성 안 함.

## 4. NpcDef / TrainerDef 변환 형식 빠른 참조

### 4.1 일반 NPC 변환

원본:
```asm
object_event 13, 20, SPRITE_YOUNGSTER, WALK, ANY_DIR, TEXT_VIRIDIANCITY_YOUNGSTER1
```
우리 `MAP_VIRIDIAN.npcs[]`:
```cpp
{13, 20, {
    L"청년: 안녕! 풀숲엔 약한 포켓몬뿐이야.",
    L"청년: 강한 포켓몬은 더 멀리 가야 만나.",
    nullptr,
}, NPC_SPR_LITTLE_BOY},   // SPRITE_YOUNGSTER → 어린 소년 대용
```

> **참고**: 현재 우리 `NpcDef` 구조에는 이동(WALK / STAY) / 방향(LEFT / RIGHT / ANY_DIR) 필드가 없음.
> 모든 NPC가 정지 상태로 렌더링됨. 이동 기능을 추가하려면 `NpcDef`에 필드 추가 + `Overworld::update()`
> 에서 NPC 이동 로직 구현 필요 (현재 범위 외).

### 4.2 트레이너 변환

원본:
```asm
object_event 30, 33, SPRITE_YOUNGSTER, STAY, LEFT, TEXT_VIRIDIANFOREST_YOUNGSTER2, OPP_BUG_CATCHER, 1
```
우리 `MAP_VIR_FOREST.trainers[]`:
```cpp
{
    30, 33,                              // x, y
    2,                                   // dir: 0=down 1=up 2=left 3=right
    4,                                   // sightRange: 시야 거리 (보통 3~5)
    L"벌레잡이 소년",                    // 트레이너 이름
    L"벌레잡이: 우리 벌레 포켓몬 한 번 봐!", // 배틀 전 대사
    {10, 10, 0},                          // partyIds (캐터피 L6 × 2 = species 10)
    {6, 6, 0},                            // partyLevels
    false,                                // defeated 플래그 (런타임)
    NPC_SPR_LITTLE_BOY,                   // OW 스프라이트
    4,                                    // 배틀 풀바디 introSpriteId (BUG_CATCHER=4)
    false,                                // isBoss
},
```
