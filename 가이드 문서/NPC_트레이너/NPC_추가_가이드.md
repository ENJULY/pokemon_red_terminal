# NPC 및 트레이너 배치 가이드

> 코드 안 읽고 NPC/트레이너만 추가하고 싶은 분을 위한 문서.
> 5분이면 첫 NPC 추가 가능. 트레이너도 비슷하지만 배틀 데이터가 추가됨.


## NPC vs 트레이너 차이

| 구분 | NPC | 트레이너 |
|------|-----|----------|
| 상호작용 | Z(A키)로 대화만 | 시야 범위에 들어가면 자동 배틀 |
| 배틀 | 없음 | 있음 (포켓몬 1~3마리) |
| 위치 차단 | 자기 칸 점유 | 자기 칸 점유 + 시야 검사 |
| 데이터 | `npcs[]` 배열 | `trainers[]` 배열 |
| 추가 필드 | 대사, 외형 | + 시야방향/거리, 파티, 풀바디 스프라이트 |
| 체육관 보스 | 불가 | `isBoss=true` 로 표시 |


## 어디에 추가하나?

**`src/data/map_data.h`** 파일 하나만 건들면 됨.

각 맵은 `inline MapDef MAP_N = { ... }` 형식으로 정의돼 있고, 그 안에 `npcs` / `trainers` 배열이 있어. 거기에 항목을 추가하면 끝.

| MAP_N | 이름 | 한국어 |
|-------|------|--------|
| MAP_0 | Pallet | 팔레트시티 |
| MAP_1 | Route 1 | 1번도로 |
| MAP_2 | Viridian | 상록시티 |
| MAP_3 | Route 2 | 2번도로 |
| MAP_4 | Viridian Forest | 상록숲 |
| MAP_5 | Pewter | 회색시티 |
| MAP_6 | Oak's Lab | 오박사 연구소 |
| MAP_7 | Player House | 주인공의 집 |
| MAP_8 | Pewter Gym | 회색시티 체육관 |
| MAP_9 | Player Bedroom | 주인공의 방 |
| MAP_10 | Rival House | 라이벌의 집 |
| MAP_11 | Viridian PC | 상록 포켓몬센터 |

# 1부: NPC 추가


## NpcDef 구조

NPC 한 명은 이 형태로 추가됨:

```cpp
{x, y, {
    L"첫 번째 대사",
    L"두 번째 대사",
    L"세 번째 대사",
    nullptr  // 끝 표시 - 항상 마지막에 nullptr
}, NPC_SPR_GENTLEMAN, 0},
```

각 항목 의미:

| 위치 | 이름 | 설명 |
|------|------|------|
| 1번째 | x | 가로 좌표 (왼쪽 = 0) |
| 2번째 | y | 세로 좌표 (위 = 0) |
| 3번째 | lines[4] | 대사 (최대 3줄 + nullptr) |
| 4번째 | spriteId | NPC 외형 (아래 표 참고) |
| 5번째 | trigger | 대화 끝나면 발동될 이벤트 (보통 0) |

**중요:**
- 모든 한글 대사는 `L"..."` 형태로 (앞에 L 붙임 - wide char)
- 대사는 **최대 3줄까지** (`lines[4]`는 3줄 + nullptr)
- 마지막에 반드시 `nullptr` 넣어줘야 함


## 사용 가능한 외형 (spriteId)

```
0  NPC_SPR_MOM          엄마 (단발 머리 어른 여자)
1  NPC_SPR_OAK          오박사 (백발 노인 + 가운)
2  NPC_SPR_BLUE         블루/라이벌 (보라 머리 남자아이)
3  NPC_SPR_GRAMPS       할아버지 (수염 노인) - 길 막기 NPC로 자주 사용
4  NPC_SPR_NURSE        간호사 조이 (분홍 머리)
5  NPC_SPR_CLERK        점원 (안경 쓴 어른)
6  NPC_SPR_GIRL         일반 여자 (긴 머리)
7  NPC_SPR_BRUNETTE     갈색머리 소녀
8  NPC_SPR_LITTLE_BOY   꼬마 남자아이
9  NPC_SPR_LITTLE_GIRL  꼬마 여자아이
10 NPC_SPR_FISHER       낚시꾼
11 NPC_SPR_GENTLEMAN    신사 (정장 남자) - 기본 트레이너 룩
12 NPC_SPR_POKE_BALL    바닥에 떨어진 몬스터볼
```


## NPC trigger 종류

대화 끝나면 발동될 특수 이벤트:

| 값 | 의미 | 사용처 |
|----|------|--------|
| 0 | 없음 | 일반 NPC (그냥 대화만) |
| 1 | STARTER_TRIGGER | 오박사 연구소 대화 → 스타터 선택 화면 |
| 2 | NURSE_HEAL | 포켓몬센터 간호사 → 파티 전원 회복 |
| 3 | MART_PARCEL | 상록 마트 점원 → 소포 받음 표시 |

**보통 NPC는 trigger=0**. 1, 2, 3은 게임 시나리오에 묶인 특수 트리거라 일반 NPC에 쓰면 안 됨.

# 2부: 트레이너 추가


## TrainerDef 구조

트레이너 한 명은 이 형태로 추가됨:

```cpp
{x, y, dir, sightRange,
 L"트레이너 이름",
 L"배틀 시작 대사",
 {pokemon1_id, pokemon2_id, pokemon3_id},
 {pokemon1_level, pokemon2_level, pokemon3_level},
 false,                       // defeated: 처음엔 false
 NPC_SPR_BRUNETTE,             // 오버월드 외형
 4,                            // 배틀 풀바디 스프라이트 ID
 false},                       // isBoss: 체육관 관장만 true
```

각 필드:

| 위치 | 필드 | 설명 |
|------|------|------|
| 1, 2 | x, y | 트레이너 좌표 |
| 3 | dir | 시야 방향 (0=↓, 1=↑, 2=←, 3=→) |
| 4 | sightRange | 시야 거리 (보통 3~5칸) |
| 5 | name | 트레이너 이름 (배틀 화면에 표시) |
| 6 | preBattleText | 배틀 시작 시 대사 |
| 7 | partyIds[3] | 포켓몬 종족 ID (없으면 0) |
| 8 | partyLevels[3] | 각 포켓몬 레벨 (없으면 0) |
| 9 | defeated | `false` (배틀 후 자동으로 true) |
| 10 | spriteId | 오버월드 외형 (NPC와 동일 표) |
| 11 | introSpriteId | 배틀 시 풀바디 스프라이트 |
| 12 | isBoss | 체육관 관장 = `true`, 일반 = `false` |


## 시야 방향 (dir)

플레이어가 시야 안에 들어오면 자동으로 배틀이 시작됨:

```
dir = 0 (↓ 남쪽):  트레이너 아래쪽으로 sightRange 칸 검사
dir = 1 (↑ 북쪽):  트레이너 위쪽
dir = 2 (← 서쪽):  트레이너 왼쪽
dir = 3 (→ 동쪽):  트레이너 오른쪽
```

예: `dir=2, sightRange=4` 트레이너가 (10, 5)에 있으면
- 플레이어가 (6, 5), (7, 5), (8, 5), (9, 5) 중 어디에 들어와도 배틀

| 11 | 메타포드 | Metapod | 벌레 | 굳히기만 |
| 13 | 뿔충이 | Weedle | 벌레/독 | 약함 |

## 배틀 풀바디 스프라이트 (introSpriteId)

배틀 시작할 때 화면에 크게 보이는 트레이너 그림:

| ID | 그림 |
|----|------|
| 0 | OAK (오박사) |
| 1 | RIVAL (라이벌/블루) |
| 2 | RED (주인공) |
| 3 | BROCK (브록) |
| 4 | BUG_CATCHER (벌레잡이 소년) |
| 5 | COOLTRAINER_M (주니어 트레이너 남) |

**없는 ID 쓰면 OAK 풀바디로 폴백**. 새로 추가하려면 `tools/gen_sprites.py`의 `INTRO_TARGETS` 리스트에 PNG 추가 + 재실행.

# 3부: 작업 체크리스트


## NPC 추가 체크리스트

- [ ] `map_data.h` 열기
- [ ] 추가할 `MAP_N` 찾기
- [ ] `npcs:` 배열 안에 새 항목 추가
- [ ] `}, N,  // npcs` 의 N 숫자 +1
- [ ] **L"..."** 형식 + 마지막 **nullptr**
- [ ] **build.bat** 실행


## 트레이너 추가 체크리스트

- [ ] `map_data.h` 열기
- [ ] 추가할 `MAP_N` 찾기 (보통 야외 도로/체육관)
- [ ] `trainers:` 배열 안에 새 항목 추가
- [ ] `}, N,  // trainers` 의 N 숫자 +1
- [ ] **partyIds[3]** + **partyLevels[3]** 같이 크기 3 배열
  - 1마리만 쓸 때도 `{16, 0, 0}, {5, 0, 0}` 형태 (나머지 0)
- [ ] **dir** 0/1/2/3 중 하나 (트레이너가 바라볼 방향)
- [ ] **sightRange** 보통 3-5 (몇 칸 앞까지 볼지)
- [ ] **defeated**: `false` (반드시)
- [ ] **isBoss**: 보스면 `true`, 아니면 `false`
- [ ] **introSpriteId**: 위 표에서 골라서 0-5 중 선택
- [ ] **build.bat** 실행



# 4부: 자주 하는 실수

### 1. NPC/트레이너가 화면에 안 보임
→ `}, N,  // npcs` 의 N 숫자를 안 늘림.

### 2. 빌드 오류 "expected '}'"
→ 대사 끝에 `nullptr` 빠짐. 또는 콤마(`,`) 빠짐.

### 3. 빌드 오류 "too many initializers for 'const wchar_t* [4]'"
→ NPC: 대사 4줄 이상 (`lines[4]`는 최대 3줄 + nullptr)
→ 트레이너: `preBattleText`는 한 줄짜리 string

### 4. 트레이너 배틀 안 시작됨
→ 시야 방향 (dir) 잘못 설정. 트레이너가 등진 방향엔 시야 없음.
→ `sightRange = 0` (0이면 시야 없음, 최소 1)
→ 플레이어가 트레이너 시야 라인에 있어야 함 (대각선 X)

### 5. 트레이너 배틀에서 적 풀바디가 OAK로 나옴
→ `introSpriteId` 안 설정했거나 잘못된 ID. 0-5 중 선택.

### 6. 보스 배틀 안 트리거됨
→ `isBoss = true` 누락. 체육관 관장만 `true`로.

### 7. 트레이너 시야에 들어갔는데 그냥 지나감
→ `defeated = true` 인 경우 시야 검사 skip. 신규 트레이너는 반드시 `false`.

### 8. 한글 깨짐
→ 파일이 UTF-8로 저장돼야 함. VSCode는 자동.

### 9. 배틀 시작 시 게임 충돌
→ partyIds 중 존재하지 않는 ID 사용 (예: 5번 포켓몬은 없음)
→ 위 "포켓몬 종족 ID" 표에 있는 것만 사용



# 5부: 좌표 및 대사 작성 팁


## 좌표 찾기

타일 한 줄을 보면 글자 하나 = 1 칸:
```cpp
"!!!)!!!!!!%%!!!!++)!"
//  ^------------^
//  cols 0-19 (20 wide map)
```

위 줄이 5번째 줄이면, `+` 가 있는 자리는 (16, 4) 와 (17, 4).

**확인 방법:**
1. `map_data.h`에서 해당 맵 찾기
2. 타일 문자열을 위에서 아래로 세서 y 찾기 (0부터)
3. 그 줄에서 왼쪽에서 오른쪽으로 글자 세서 x 찾기 (0부터)

게임 내 **Ctrl+M** 디버그 워프로 원하는 맵 점프해서 실제 위치 가늠 가능.


## 대사 작성 팁

좋은 NPC/트레이너 대사:
- 짧고 명확 (한 줄 30자 이내 권장)
- 캐릭터 분위기 반영 (`할아버지:`, `여자:`, `꼬마:` 등 화자 명시)

원본 포켓몬 레드 한국어판 톤 참고:
```
"할아버지: 좋아! 이 길은 통과하면 안 된다네."
"벌레잡이 소년: 이봐! 너 포켓몬 트레이너지!"
"여자: 친구가 새로운 포켓몬을 잡았어!"
"꼬마: 우리 아빠는 강해!"
"관장 브록: 나는 바위처럼 단단한 수비를 믿어!"
```


## 트레이너 시야 설계 팁

- **좁은 통로**: dir = 시야 방향, range = 통로 길이
- **체육관 보스**: range = 4~5 (홀 전체 커버하도록)
- **숨어 있는 트레이너**: 풀숲(`;`) 옆에 배치하면 잘 안 보임
- **트레이너 라인 막기**: 시야 라인에 벽이 있어도 시야는 통과 (구현 한계, 직선 시야만)
