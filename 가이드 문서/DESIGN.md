# 포켓몬스터 레드 - 터미널 C++ 구현 기획서
> 목표: 1관장(브록) 클리어까지 플레이 가능한 터미널 게임

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 언어 | C++ (C++17) |
| 렌더링 | 터미널 + ANSI 이스케이프 코드 |
| 그래픽 | 실제 소스 이미지 → ASCII Art 변환 후 사용 |
| 플랫폼 | Windows (CMD / Windows Terminal) |
| 목표 범위 | 팔레트시티 ~ 회색시티 1관장(브록) 클리어 |

## 3. 그래픽 처리 방식

### ASCII Art 변환 파이프라인

```
원본 이미지 (PNG/BMP)
       ↓
  전처리 (크기 조정, 팔레트 축소)
       ↓
  ASCII 변환 (밝기 → 문자 매핑)
       ↓
  ANSI 컬러코드 적용
       ↓
  .h 파일로 하드코딩 저장
```

### 사용할 ASCII 문자 셋
```
밝기 순 (어두움 → 밝음):
█ ▓ ▒ ░ # @ % * + = - : . ' ' (공백)
```

### 적용 대상
- 포켓몬 배틀 스프라이트 (앞모습 / 뒷모습)
- 타일셋 (맵 타일: 풀, 건물, 나무, 물 등)
- 플레이어 캐릭터 (4방향 이동 스프라이트)
- UI 프레임 (배틀창, 대화창, 메뉴창)

## 6. 기술 스택 & 라이브러리

| 역할 | 선택 | 이유 |
|------|------|------|
| 터미널 제어 | Windows API + ANSI | 외부 의존 없음 |
| 키입력 | `<conio.h>` (_kbhit, _getch) | Windows 표준 |
| 빌드 | CMake + MinGW (g++) | 설치되어 있음 |
| 이미지 변환 도구 | stb_image.h (single header) | 경량, 헤더만 추가 |

## 7. 데이터 설계

### 포켓몬 구조체
```cpp
struct PokemonBase {
    int id;
    string name;
    Type type1, type2;
    int baseHP, baseAtk, baseDef, baseSpe, baseSpc;
    vector<LearnMove> learnset; // 레벨업 기술
};

struct Pokemon {
    PokemonBase* base;
    int level;
    int currentHP, maxHP;
    Move moves[4];
    int exp;
    // 실제 스탯 (레벨 반영)
    int hp, atk, def, spe, spc;
};
```

### 맵 타일
```cpp
enum TileType {
    TILE_GRASS,       // 이동 가능, 야생 인카운터
    TILE_TALL_GRASS,  // 야생 포켓몬 출현
    TILE_WALL,        // 충돌
    TILE_TREE,        // 충돌
    TILE_WATER,       // 충돌 (나중에 파도타기)
    TILE_DOOR,        // 건물 진입
    TILE_PATH,        // 이동 가능
    TILE_NPC,         // NPC 위치
};
```

### 타입 상성 테이블
```cpp
// 18x18 float 테이블
// type_chart[공격타입][방어타입] = 배율 (0, 0.5, 1, 2)
float type_chart[NUM_TYPES][NUM_TYPES];
```

## 8. 게임 루프

```
while (게임 실행 중) {
    1. 입력 처리 (논블로킹)
    2. 씬 업데이트
       - SCENE_INTRO    → 오박사 인트로
       - SCENE_OVERWORLD → 맵 이동 / NPC / 이벤트
       - SCENE_BATTLE   → 턴제 배틀
       - SCENE_MENU     → 포켓몬 목록 / 아이템 / 저장
    3. 렌더링 (더블버퍼 diff 렌더)
    4. 프레임 대기 (약 60ms = ~16fps)
}
```
