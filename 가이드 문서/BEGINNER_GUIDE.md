# Pokemon Red 터미널 - 초보자도 이해하는 설명

> **누가 읽어야 하나?**
> "C++? 그게 뭐야?" 정도의 코딩 입문자. 이 문서는 *어떻게 작동하는지의 큰 그림*을 친구한테 설명하듯 풀어둔 거야. 코드를 직접 짤 줄 몰라도 OK.


## 이 프로젝트가 뭐야?

**검정 터미널 창**(명령 프롬프트, cmd.exe)에서 옛날 게임보이 포켓몬 레드를 그대로 돌리는 프로젝트.

보통 게임은 그래픽 카드가 화면에 픽셀을 그리지만, 이 프로젝트는 **글자만 가지고** 그림을 그려. 검정 네모, 회색 네모, 흰 네모 같은 색깔 있는 글자를 적절히 배치하면 픽셀처럼 보이거든.

```
ㅁㅁㅁ▓▓▓ㅁㅁㅁ      <- 글자 6개를 일렬로 놓으면
ㅁ▓▓ㅁㅁ▓ㅁㅁㅁ      <- 그림처럼 보임
ㅁㅁ▓▓▓▓ㅁㅁㅁ
```

그래서 **그래픽 카드 없이도** 옛날 게임의 모습을 흉내낼 수 있는 거야.


## 핵심 비유: 도시락 만들기

게임이 화면을 그리는 과정을 도시락 만들기에 비유해볼게.

### 1. 재료 (Pokemon 원본 그래픽)

옛날 게임보이 게임 데이터는 인터넷에 공개돼 있어 (`../pokered/` 폴더). 거기엔:
- 픽셀 그림 (PNG 이미지)
- 어디에 어떤 그림을 놓을지 적은 표 (`.blk`, `.bst` 파일들)

이건 "냉장고 안의 식재료"야.

### 2. 요리사 (Python 스크립트 = 자동 변환 도구)

`tools/` 폴더 안의 `.py` 파일들이 요리사 역할이야. 이 친구들이 하는 일:

> "PNG 그림 한 장을 작은 조각으로 자른다 → 각 조각마다 글자 한 개를 정해준다 → 그 글자를 화면에 출력하면 그림처럼 보이게 ANSI 코드를 만들어준다"

예를 들어:
- 풀밭 그림 8x8 픽셀 → 문자 `!` 로 표현
- 나무 그림 → 문자 `$`
- 도어 그림 → 문자 `D`

그래서 **지도 한 줄이 그냥 문자열**이 돼:
```
"!!!)!!!!!);;)!!!!!)!"
```
이게 사실 풀밭, 길, 풀숲이 섞인 지도 한 줄이야.

### 3. 도시락통 (C++ 자동생성 파일들)

요리사(Python)가 만든 결과물은 `src/data/` 폴더에 저장돼:
- `tiles.h` - 모든 작은 그림 조각의 모양 (글자로 표현된 그림)
- `sprites.h` - 캐릭터/NPC 모양
- `map_data.h` - 각 지도가 어떤 글자들로 구성됐는지

이건 한 번 만들어두면 끝. 게임 켤 때마다 새로 안 만들어.

### 4. 점원 (게임 엔진 = C++ 코드)

`src/engine/`, `src/game/` 안의 C++ 코드가 점원 역할:
- 너가 키보드를 누르면 → 캐릭터 위치 업데이트
- 매 1/60초마다 → 현재 위치에서 보이는 도시락(타일들)을 골라서 화면에 출력
- 풀숲 밟으면 → 야생 포켓몬 등장
- NPC 앞에서 Z누르면 → 대화창


## 게임이 한 순간에 뭘 하는가? (메인 루프)

게임이 켜져 있는 동안 컴퓨터는 **1초에 약 60번** 이 작업을 반복해:

```
[1/60초마다 반복]
  1. 화면을 깨끗하게 지운다
  2. "지금 어느 장면이지?" 확인 (오프닝? 마을? 배틀?)
  3. 그 장면에 맞는 그림을 그린다
  4. 한국어 텍스트를 위에 덧칠한다
  5. 변한 부분만 실제 터미널 창에 출력한다 (성능을 위해)
  6. 키보드 입력 확인 (방향키, Z, X 등)
  7. 입력에 따라 캐릭터 움직이거나 메뉴 선택
[다시 1번으로]
```

이게 영화 필름이 1초에 24장씩 넘어가서 움직이는 것처럼 보이는 원리야.


## "장면"이 뭐야?

게임은 큰 흐름으로 보면 **여러 장면이 이어진 연극**이야:

```
오프닝 (오박사 등장)
   ↓
이름 입력
   ↓
라이벌 이름 입력
   ↓
연구소 (스타터 선택)
   ↓
야외 탐험 <- 가장 많은 시간 보내는 장면
   ↓
배틀 (야생/트레이너/관장)
   ↓
탐험으로 복귀
   ...
   ↓
엔딩
```

코드에서는 `Scene` 이라는 이름으로 관리해 (`game.h`):
```
INTRO, OVERWORLD, WILD_BATTLE, TRAINER_BATTLE, ...
```

각 장면마다 "이 장면일 때 어떻게 보이고, 키 누르면 뭘 하는지"가 따로 정의돼 있어.

## 지도는 어떻게 그려져?

### 지도 한 줄 = 문자열 한 줄

지도 한 칸은 **글자 한 개**로 표현돼. 예를 들어 팔레트시티의 한 줄은:
```
")!(&(&(&(&(&(&(&(&!)"
```

각 글자의 뜻:
- `)` = 벽이나 울타리 (지나갈 수 없음)
- `!` = 풀밭 (지나갈 수 있음)
- `(`, `&` = 다른 종류의 풀밭
- `;` = 긴 풀숲 (야생 포켓몬 나올 수 있음)

### 글자 → 그림 변환

게임이 화면을 그릴 때:
1. 너의 현재 위치 기준으로 주변 글자들을 가져옴
2. 각 글자마다 "이건 어떻게 생긴 그림이지?" 확인 (tiles.h가 답 알려줌)
3. 그 그림을 화면 적절한 위치에 출력

### 같은 글자라도 장소에 따라 모양 다름

이게 살짝 신기한데, `!` 라는 글자가:
- **밖** (마을, 도로)에서는 → 풀밭으로 그려짐
- **체육관** 안에서는 → 체육관 바닥으로 그려짐
- **상점** 안에서는 → 상점 바닥으로 그려짐

같은 글자지만 "지금 어느 지도?"를 따져서 다른 그림으로 출력하는 거지. 그래서 글자 종류가 적어도 다양한 장소를 표현할 수 있어.


## 캐릭터 움직임

너가 → 키를 누르면:
1. 컴퓨터: "어, 오른쪽 누름. 캐릭터 위치를 한 칸 오른쪽으로 옮길 수 있나?"
2. 한 칸 오른쪽의 글자가 뭔지 확인 (`!` 풀밭? `$` 나무?)
3. 통과 가능한 글자면 → 위치 변경 + 걷는 애니메이션 4프레임 재생
4. 막힌 글자(나무, 벽)면 → 방향만 바꾸고 안 움직임
5. 도어(`D`) 글자면 → "워프 트리거!" → 다른 지도로 페이드 전환

### 절벽 점프

`0`, `5`, `6`, `<` 글자는 절벽이야. 위에서 ↓ 누르면 점프해서 두 칸 아래로 내려가. 반대로 올라올 순 없어. 옛날 게임의 절벽 같은 거.

### 풀숲 인카운터

`;` 글자(긴 풀숲) 위를 걸을 때마다 15% 확률로 "야! 야생 포켓몬이 나타났다!" 이벤트가 발동돼서 배틀 장면으로 넘어가.


## 배틀은 어떻게 돌아가?

배틀도 "장면"이야. 야외 탐험에서 배틀로 넘어가면:

```
1. INTRO_TRANSITION  적 등장 애니메이션
2. SHOW_MSG          "야생 꼬렛이 나타났다!"
3. CHOOSE_COMMAND    싸운다 / 가방 / 교체 / 도망 선택
4. CHOOSE_MOVE       4기술 중 하나 선택
5. PLAYER_ACTION     데미지 계산 + 메시지
6. ENEMY_ACTION      상대도 공격
7. CHECK_FAINT       누가 기절했는지 확인
8. GRANT_EXP/LEVEL_UP 경험치/레벨업
9. DONE              배틀 끝, 야외로 복귀
```

데미지 계산은 옛날 1세대 포켓몬 공식 그대로:
```
데미지 = (공격력 × 기술위력 / 방어력) 계산 × 타입상성 × 보정
```


## 자동 생성 파이프라인

요리사(Python)가 하는 일을 좀 더 자세히 보면:

```
   원본 자료 (pokered)
        |
  [ 1. PNG 그림 ]   <- 8x8 픽셀 작은 조각으로 자름
        |
  [ 2. 조각 매핑 ]  <- 각 조각마다 글자 1개 배정
                     (자주 나오는 조각 = 짧고 흔한 글자)
        |
  [ 3. ANSI 변환 ]  <- 검정/회색/흰색 픽셀을
                     색깔 ANSI 코드로 바꿈
        |
  [ 4. 지도 생성 ]  <- 원본 .blk 파일 읽으면서
                     글자 줄줄이 만들어냄
                     "!!!)0000<$5656%%%%)!"
        |
   C++ 헤더 파일 자동 생성
   (tiles.h, map_data.h)
        |
   g++ 컴파일러
        |
   build/PokemonRed.exe
```

이걸 사람이 손으로 했으면 18개 지도, 수백 개 조각을 다 그렸겠지만, 스크립트가 해주니까 1초도 안 걸려.


## 그림은 결국 어떻게 화면에 보이나?

옛날 컴퓨터에서 글자에 색을 입히는 방법 ("ANSI 이스케이프 코드")을 써:

```
\x1b[48;5;232m \x1b[0m
└──────┬──────┘└──┘
   이거: "검정 배경"   "원래대로"
```

이 `\x1b[48;5;232m ` 라는 문자열을 출력하면 터미널이 "아 다음 글자부터는 검정 배경으로 그려야겠다" 하고 알아채는 거야.

그리고 `▀` 라는 특수 문자는 "위쪽 절반만 색칠된 사각형"이야. 위와 아래를 다른 색으로 칠해서 2픽셀처럼 보이게 할 수 있어. 그래서 8글자 x 8행이면 16픽셀처럼 보이는 거지.

```
글자 한 칸 = 8 x 16 픽셀 (실제로는 글자 폭에 따라 다름)
타일 한 개 = 16글자 x 8행 = 16 x 16 픽셀 같은 효과
```


## 게임이 만들어진 순서 (개발 흐름)

1. **원본 그래픽 확보** - pokered 디스어셈블리 프로젝트에서 그림/지도 데이터 가져옴
2. **Python으로 자동 변환** - 그림을 글자+ANSI 코드로 바꾸는 스크립트 작성
3. **C++ 엔진 작성** - 매 프레임 화면 그리고 키 입력 받는 기본 구조
4. **지도 탐험 구현** - 캐릭터 움직임, 절벽 점프, 워프
5. **NPC/대화 시스템** - A 키로 대화, 카운터 너머 대화
6. **배틀 시스템** - 턴제 전투, 데미지 공식, AI
7. **씬 전환** - 오프닝부터 엔딩까지 흐름 연결
8. **버그 수정** - 사용자 테스트하면서 발견되는 이슈들 (지금 단계)


## 자주 나올 법한 질문

**Q. 왜 글자로 그림을 그려?**
A. 그래픽 카드 없이도 동작하니까. 검정 cmd 창에서 옛날 게임 분위기 살리는 게 목표였어.

**Q. 사운드는?**
A. `winmm.lib` 라는 윈도우 기본 라이브러리로 WAV 파일 재생 가능. 지금은 기본 골격만 있어.

**Q. 다른 OS에서도 돌아가?**
A. 지금은 Windows 전용 (Win32 콘솔 API 사용). 리눅스/Mac에 옮기려면 input/audio 부분 다시 짜야 해.

**Q. 손으로 지도를 그릴 수 있어?**
A. 가능. `map_data.h` 파일에서 글자 줄을 수정하면 돼. 다만 Python 스크립트 다시 돌리면 자동생성된 부분은 덮어써짐.

**Q. 포켓몬을 추가하려면?**
A. `src/data/pokemon_data.h`에 종족값/기술 추가. 그리고 `src/data/sprites.h`에 그림 추가 (요건 Python으로 생성).

**Q. C++ 배워야 하나?**
A. 이 프로젝트를 이해만 하려면 안 배워도 OK. 직접 수정하려면 기본 문법 (변수, 함수, if/for) 정도는 알아야 편함.
