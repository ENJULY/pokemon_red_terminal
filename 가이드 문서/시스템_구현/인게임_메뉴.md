## 1. 인게임 메뉴 시스템 (M 키)

### 어디에 있나

| 파일 | 역할 |
|------|------|
| `src/game/game.h` | `Scene::INGAME_MENU` enum + `InGameMenuState` enum + 상태 변수 |
| `src/game/game.cpp` | `updateInGameMenu()`, `renderInGameMenu()`, `renderInGameMenuKorean()` |

### 동작 흐름

```
오버월드 중 M 키
  └→ Game::update()
       └→ prevScene_ 저장, menuState_ = TOP_LEVEL
          changeScene(Scene::INGAME_MENU)

INGAME_MENU 상태
  TOP_LEVEL  ──[A]→  PARTY_VIEW  ──[A]→  PARTY_DETAIL
                └─→  ITEM_STUB
                └─→  SAVE (타이머 메시지)
  모든 상태 ──[BS(Backspace)]→ 한 단계 위 / 오버월드 복귀
```

### 씬 구조 — `Scene` enum (`game.h`)

`Scene::INGAME_MENU` 는 기존 `Scene::OVERWORLD` 위에 덮는 방식이 아니라
완전히 별도 씬으로 전환됩니다. 복귀 시 `prevScene_` 에 저장해둔 씬으로 돌아갑니다.

```cpp
// game.h
enum class InGameMenuState {
    TOP_LEVEL,
    PARTY_VIEW,
    PARTY_DETAIL,
    ITEM_STUB,
};

int          menuCursor_     = 0;
int          partyMenuCursor_= 0;
int          detailPartyIdx_ = 0;
Scene        prevScene_;
bool         menuSaveMsg_    = false;
int          menuSaveMsgTimer_= 0;
```

### 렌더링 구조 — 더블 레이어

`render()` → `renderKorean()` 순서로 호출됩니다.
- `renderInGameMenu()` : ASCII 박스, 수치(숫자), HP바 출력
- `renderInGameMenuKorean()` : 한글 라벨 출력 (ASCII 위에 덮어씀)

이 방식 덕분에 한글과 ASCII를 다른 색/위치로 관리할 수 있습니다.

## 2. 포켓몬 상세 화면 — 기술/상성 정보

### 레이아웃 (62×24 박스, by=2)

```
┌──────────────────── [ 포켓몬 상세 ] ────────────────────────┐
│  이상해씨                                                    │
│  풀 / 독                                                    │
│                                                             │
│  HP 45/45  Lv.5          ← 한글 "HP" 가 ASCII "Lv." 를 덮음 │
│                                                             │
│  공격 49   방어 49        ← 한글 라벨 + ASCII 수치           │

│                                                             │
│  ── 기술 ──                                                 │

│  울음소리  노말  PP:40/40  PWR: --  ACC:100                │
│  (없음)                                                     │
│  (없음)                                                     │
│                                                             │
│  ── 상성 ──                                                 │
│  약점(2x) : 불꽃 얼음 에스퍼 비행                           │  ← 빨간색
│  저항(½x) : 물 전기 풀 격투                                │  ← 초록색
│  무효(0x) : 없음                                           │  ← 회색
│                                                             │
│                                                             │
│                                                             │
│  [Z]: 뒤로                                                  │
└─────────────────────────────────────────────────────────────┘
```

### 타입 상성 계산 (`type_chart.h`)

```cpp
// 12개 공격 타입 전체에 대해 getEffInt() 호출
for (int ti = 0; ti < 12; ti++) {
    Type atk = (Type)ti;
    int eff = getEffInt(atk, def1, def2);  // 0/5/10/20 반환
    if (eff == 0)  → 무효 목록에 추가
    if (eff >= 20) → 약점 목록에 추가
    if (eff <= 5)  → 저항 목록에 추가
}
```

`getEffInt(Type atk, Type def1, Type def2)` 는 이중 타입 방어를 처리합니다.
예: 풀/독 타입 → 불꽃 공격은 풀에 2x, 독에 1x → 결과 2x (20).

### 기술 정보 출력

`MoveData` 구조체 (`pokemon_data.h`) 에 `power`, `accuracy` 필드가 있습니다.
- `power == 0` → 변화기술 → `PWR: --` 로 표시
- `power > 0` → `PWR: xxx` 로 표시

```cpp
// game.cpp renderInGameMenu() PARTY_DETAIL 블록
if (mv.power > 0)
    snprintf(..., "PP:%2d/%2d  PWR:%3d  ACC:%3d", ...);
else
    snprintf(..., "PP:%2d/%2d  PWR: --  ACC:%3d", ...);
renderer.print(bx+28, by+11+i, mvbuf, ...);  // ASCII 수치
renderer.printW(bx+4,  by+11+i, mv.name, ...); // 한글 기술명 (renderKorean)
renderer.printW(bx+20, by+11+i, menuTypeName(mv.type), ...); // 타입명
```
