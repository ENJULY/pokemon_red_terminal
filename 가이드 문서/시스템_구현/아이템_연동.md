# 아이템 시스템 연동 가이드

아이템 시스템을 개발한 뒤 인게임 메뉴의 **가방(ITEM_STUB)** 화면과 연동하는 방법을 안내합니다.

---

## 1. 현재 상태

### 아이템 데이터 위치
`src/game/player.h` — `Player` 구조체 내부

```cpp
struct Player {
    // 현재 아이템 필드 (단순 카운터)
    int  pokeballs;    // 몬스터볼
    int  superballs;   // 슈퍼볼
    int  potions;      // 포션
    int  superpotions; // 슈퍼포션
    bool hasPokedex;
    ...
};
```

### 메뉴 스텁 위치
`src/game/game.cpp` — `InGameMenuState::ITEM_STUB`

- `Game::updateInGameMenu()` — `case InGameMenuState::ITEM_STUB:` (Z키 → TOP_LEVEL 복귀만 처리)
- `Game::renderInGameMenu()` — `case InGameMenuState::ITEM_STUB:` (수량 ASCII 출력)
- `Game::renderInGameMenuKorean()` — `case InGameMenuState::ITEM_STUB:` (아이템명 한글 출력)

## 2. 아이템 시스템 설계 권장 사항

### 2-A. `Player` 구조체 확장 (player.h)

단순 카운터 대신 배열+enum 방식을 권장합니다.

```cpp
// player.h에 추가
enum class ItemId {
    NONE = 0,
    MONSTER_BALL,
    SUPER_BALL,
    MASTER_BALL,
    POTION,
    SUPER_POTION,
    ANTIDOTE,
    // 추가 아이템...
};

struct ItemSlot {
    ItemId id;
    int    count;
};

struct Player {
    // 기존 필드 유지 또는 아래로 교체
    ItemSlot bag[20];   // 최대 20종
    int      bagSize;
    ...
};
```

> **주의**: 기존 `pokeballs`, `superballs`, `potions`, `superpotions` 필드를 삭제하면
> `game.cpp`의 마트 이벤트(`updateMart`), 포켓덱스 수령(`updateReceiveDex`),
> 워프 메뉴 디버그 지급 코드도 함께 수정해야 합니다.
> 검색: `grep -n "pokeballs\|potions\|superpotions\|superballs" src/game/game.cpp`

## 3. 연동 포인트

### 3-A. 가방 화면 렌더링 교체

**파일**: `src/game/game.cpp`

`renderInGameMenu()` 내 `case InGameMenuState::ITEM_STUB:` 블록을 교체합니다.

```cpp
// 교체 전 (스텁)
case InGameMenuState::ITEM_STUB: {
    // ...단순 수량 출력...
}

// 교체 후 (아이템 시스템 연동)
case InGameMenuState::ITEM_STUB: {
    int bw = 44, bh = 14, bx = W/2 - bw/2, by = 5;
    renderer.drawBox(bx, by, bw, bh, box);
    renderer.fillRect(bx+1, by+1, bw-2, bh-2, ' ', fill);
    // 커서 표시
    for (int i = 0; i < player_.bagSize; i++) {
        bool sel = (i == itemMenuCursor_);
        renderer.print(bx+2, by+3+i, sel ? ">" : " ", sel ? hl : fill);
        // 수량
        char cnt[8]; snprintf(cnt, sizeof(cnt), "x%d", player_.bag[i].count);
        renderer.print(bx+36, by+3+i, cnt, sel ? hl : fill);
    }
    break;
}
```

`renderInGameMenuKorean()` 내 `case InGameMenuState::ITEM_STUB:` 블록에서
아이템 이름을 동적으로 출력합니다.

```cpp
case InGameMenuState::ITEM_STUB: {
    // ...
    for (int i = 0; i < player_.bagSize; i++) {
        bool sel = (i == itemMenuCursor_);
        renderer.printW(bx+4, by+3+i, getItemName(player_.bag[i].id), sel ? hl : fill);
    }
    // ...
}
```

### 3-B. 아이템 사용 처리

**파일**: `src/game/game.cpp`

`updateInGameMenu()` 내 `case InGameMenuState::ITEM_STUB:` 블록을 확장합니다.

```cpp
case InGameMenuState::ITEM_STUB:
    if (key == Key::UP)   itemMenuCursor_ = (itemMenuCursor_ - 1 + player_.bagSize) % player_.bagSize;
    if (key == Key::DOWN) itemMenuCursor_ = (itemMenuCursor_ + 1) % player_.bagSize;
    if (key == Key::A) {
        // 아이템 사용 처리
        useItemOnParty(player_, itemMenuCursor_, partyMenuCursor_);
    }
    if (key == Key::B)    menuState_ = InGameMenuState::TOP_LEVEL;
    break;
```

## 5. 저장 시 아이템 데이터 보존

추후 세이브 기능 구현 시, `Player::bag[]` 배열을 직렬화할 때 포인터가 없는 POD 구조이므로
`ItemSlot` 배열을 그대로 바이너리 저장/로드할 수 있습니다.

```cpp
// 저장 예시 (추후 구현)
fwrite(player_.bag, sizeof(ItemSlot), player_.bagSize, fp);

// 로드 예시
fread(player_.bag, sizeof(ItemSlot), MAX_BAG_SIZE, fp);
```

## 6. 소포 퀘스트 보상 연동

소포 퀘스트는 현재 플래그만 세팅하고 보상을 지급하지 않습니다.
아이템 시스템 완성 후 아래 위치에 보상 코드를 추가합니다.

### 퀘스트 흐름

| 단계 | 플래그 | 설명 |
|------|--------|------|
| 1 | `gotParcel = false` | 초기 상태 |
| 2 | `gotParcel = true` | 상록 마트 첫 방문 → MART_EVENT 종료 시 세팅 (`game.cpp: updateMart`) |
| 3 | `deliveredParcel = false` | 오박사에게 아직 전달 안 함 |
| 4 | `deliveredParcel = true` | 오박사 대화 완료 시 세팅 — **보상 지급 위치** |

### 보상 지급 위치

**파일**: `src/game/overworld.cpp`

NPC 대화 종료 핸들러에서 Oak 태그 처리 부분을 찾습니다.

```cpp
// overworld.cpp — 대화 끝(Z키) 처리
if (npc->tag == NPC_TAG_OAK_LAB && useOverrideLines_) {
    if (pl_.gotParcel && !pl_.deliveredParcel) {
        pl_.deliveredParcel = true;
        // ↓ 여기에 보상 추가
        // 예시: pl_.pokeballs += 5;
        // 예시: addItemToPlayer(pl_, ItemId::POTION, 3);
    }
}
```

현재는 `pokeballs += 3`을 마트에서 임시로 지급하고 있습니다.
아이템 시스템 완성 후 마트 보상(`game.cpp: updateMart`)과 오박사 보상을 별도로 설계하세요.

### 보상 예시 (미정)

```cpp
// 오박사 소포 전달 보상 — 팀에서 결정 후 작성
// 예: 포션 3개
pl_.potions += 3;
// 예: 특정 아이템 1개 (ItemId 기반 시스템 사용 시)
// addItem(pl_, ItemId::REVIVE, 1);
```
